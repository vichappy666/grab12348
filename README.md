# 12348 抢班次工具

广东 12348 公共法律服务平台（省热线平台法服岗）值班**抢班次**脚本。
给自己抢值班班次用：到点自动盯放班、发现名额立刻抢、抢到后桌面通知 + 语音播报。

> 上次没抢到的原因，是发请求那一刻名额已经满了（接口返回 `抢班人数已超过限制！`）。
> 本工具把「盯名额 → 抢」的循环压到几十毫秒级，并支持定时卡点、并发抢多个候选。

## 工作原理

抢班只依赖两个接口：

- `GET  .../getGrabSchedulingList/{日期}` —— 查某天可抢的班次（含名额 `已抢/总数`）
- `POST .../grabScheduling/{schedulingId}` —— 抢指定班次（靠 token 鉴权，无 body）

登录那一步需要手机短信验证码（没法自动化），所以采用 **token 模式**：
你手动登录一次，把 token 复制进配置，脚本拿 token 干活。
（token 里没有过期时间，是服务端 session 引用，登录后一段时间内一直有效。）

## 放班规则

平台每周放班两次，放班时刻 **13:00**（原为 18:00）：

- **周一 13:00** 放 周二 / 周三 / 周四 的班
- **周四 13:00** 放 周五 /（周六 / 周日 不抢）/ 下周一 的班

配置里日期和开抢时刻都写 `auto`（默认已配好），脚本会自动算出
下一个放班时刻和对应可抢的日期窗口（**周六、周日自动跳过、不抢**），
平时只需要 `python main.py run`。

## 抢班优先级

默认开 **`per_day` 模式**：一次 `run` 给窗口里**每天各抢一个班**，抢到某天就锁定该天，
全部抢到即收工。每天按一张**顺位表**从高到低抢，上一顺位满了/秒光就自动落到下一顺位。

顺位表按星期分两套（在 `config.yaml` 的 `target.order` / `order_friday` 配置）：

- **周一 / 周二 / 周三 / 周四**：`C › B › D › E › F › C3 › C1 › R › G`
- **周五**：`C › B › D`（其他班次不抢）
- **周六 / 周日**：不抢

同一个班次分「优 / 普」两种场次，`target.prefer` 决定先抢哪种（默认 `优`）。
匹配是**精确到班次代码**的：顺位里写 `C` 只命中 `C`，不会误抓 `C1` / `C3`；
早高峰 / 午高峰 / 晚高峰这类不在顺位表里的班次一律不抢。

## 安装

需要 Python 3.9+。

```bash
cd grab12348
python3 -m venv .venv && source .venv/bin/activate   # 可选，推荐
pip install -r requirements.txt
```

## 怎么拿 token

1. 用手机或电脑浏览器登录 `https://gd.12348.gov.cn/cloudexamh5/login`
2. 打开开发者工具（手机可用电脑 Chrome 的「检查」/ Safari 远程调试；电脑直接 F12）
3. 切到 **Network（网络）** 面板，在页面上随便点一下排班相关操作，
   找到任意一个发往 `cloudh5api/...` 的请求
4. 看它的请求头 **Authorization**，值形如 `Bearer eyJhbGci...`，
   把 `Bearer ` 后面那一长串复制下来
5. 粘贴到 `config.yaml` 的 `auth.token`

> 提示：返回里带 `"token": "..."` 的登录响应里那串也是同一个 token。

## 配置

把 `config.example.yaml` 复制成 `config.yaml`（已自带一份），按注释填：

- `target.dates`：要抢哪天。默认 `auto`=按放班规则自动算（周六日自动跳过）；也可写死如 `2026-07-10`（可与 auto 混用）
- `target.order`：周一~周四的班次顺位（从高到低），默认 `[C, B, D, E, F, C3, C1, R, G]`；精确匹配代码，写 `C` 不会碰 `C1`/`C3`
- `target.order_friday`：周五的顺位，默认 `[C, B, D]`
- `target.prefer`：同一班次优先抢 `优` 还是 `普`（默认 `优`）
- `grab.start_at`：放班开抢时刻。默认 `auto`=自动等下一个放班时刻（周一/周四 13:00）；也可写 `13:00:00`（当天该时刻，已过顺延明天）或绝对时间 `2026-07-10 13:00:00`；留空=运行即开抢
- `grab.per_day`：默认 `true`=每个目标日期各抢一个班（抢到即锁定该天）；设 `false` 则改用 `stop_after` 只抢到 N 个就停、不区分日期
- `grab.aggressive`：秒光场景设 `true`（到点直接对已知班次盲抢，不等列表确认名额）

## 使用

```bash
# 1) 先校验 token 是否有效，看本月已排班 / 已满日期
python main.py check

# 2) 看某天有哪些可抢班次、各自还剩多少名额、对应 schedulingId
python main.py list 2026-05-22

# 3) 演练（不会真的抢）：确认筛选条件能命中你要的班
python main.py run --dry-run

# 4) 正式跑（auto 模式：自动等下一个放班时刻——周一/周四 13:00——日期也自动定）
python main.py run

# 立即开抢、忽略定时（抢当前窗口的剩余名额）：
python main.py run --now
```

抢到后会：终端响铃、弹 macOS 桌面通知、用 `say` 播报「抢到班次了」。

## 实战建议

- **先 `check` 再 `list`**：确认 token 没过期、确认目标班次名和 schedulingId 对得上。
- **start_at 用 `auto` 就不用手动对放班时刻**：脚本自动等下一个周一/周四 13:00。注意本机系统时间要准，差几秒就可能错过，可提前 `date` 对一下时间。
- **名额秒光就开 `aggressive`**：标准模式要先看到「有名额」再抢，会慢半拍；
  盲抢模式到点直接发，更适合一放出就被抢光的热门班。
- **poll_interval 别调太狠**：这是政府服务器，你只是抢自己一个班，
  0.2~0.5 秒一轮足够，没必要每秒几十发。
- **token 会随重新登录而变**：失效了 `check` 会直接告诉你，重新复制即可。
- **服务器慢是常态**：这个政府站点响应经常要好几秒、偶尔卡到 20 秒以上，`check`/`list`
  查询类请求超时会自动重试几次，所以偶发的一次超时不用管，多等一下即可。嫌不够可把
  `auth.timeout` 调大。若报「token 无效」才是真失效，那种会明确提示，跟超时是两回事。
